    Quiz-its Delish

Project overview

Quiz-its is a responsive CRUD product that users can login, select a quiz to play, get results, save and delete results. The project was complete within a two-week deadline. The requirements include developing user personas, conducting user interviews and research for insights to develop the product, creating wireframes, prototypes, using Firebase for database storage and user authentication, and GitHub project tracking. Tools and techologies used to build the application: HTML, CSS, SASS, jQuery, JavaScript, Bootstrap, Browserify, Grunt, GitHub, Extensio, Adobe XD, Photoshop, and Illustrator.

Challenge Statement

How might we create a quick and fun product for users to engage with to take a break from everyday stress.

User Research Interviews

Interviewed 7 users of various ages ranging from 20 something millenials to 50-year-old professionals that enjoy playing social media quizzes or participating in trivia game.

Research Insights

  • Users like to test their general knowledge skills.
  • Users like to keep the results to themselves and seldom share their quiz results on social media.
  • User reasons for taking a quiz or playing trivia vary from liking learning new fun facts to learning more about themselves through answering the questions.
  • Users spend about 10 minutes a day/week engaged in a social or trivia quiz.

Sketches + Wireframes

After reviewing several competitor websites such as Playbuzz, Quizony, Sporcle, and Zimbo, I mocked up a quick sketch of what the quiz product layout would look like.

Wireframes

When it came time to work on the wireframes, the designed changed based on some user feedback. Taking into consideration the feedback, it was a better resolution to display the quiz questions in a linear column rather than horizontal display of questions.

The design changed again during the site prototyping to include more interactivity to the quiz images rather than just a static image link similar to standout from the competitor sites reviewed.

Project Reflections

  • Creating quizzes from scratch take time to create and craft questions and answers. On further review and refactoring of the project, I plan to used the trivia API content to create more quizzes.
  • To improve the user experience, there needs to be more user testing to refactor content and the user interface of the project.
